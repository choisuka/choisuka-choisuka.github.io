# toro_math 프로젝트 지침

## 1. 기능 구현 후 테스트

기능 구현이 끝나면 반드시 테스트까지 완료한 뒤 보고한다.
- 브라우저에서 해당 기능을 직접 실행하여 동작 확인
- 엣지 케이스(경계값, 빈 입력 등) 포함

## 2. 작은 단위로 커밋

변경은 작은 단위로 나눠서 커밋한다.
- 하나의 커밋 = 하나의 논리적 변경 (카드 추가, 버그 수정, 스타일 조정 등)
- 여러 기능을 한 번에 커밋하지 않는다

## 3. 기술 스택

| 항목 | 내용 |
|------|------|
| 구조 | 단일 파일 앱 (`index.html`) — HTML + CSS + JS 통합 |
| 프레임워크 | 없음 (Vanilla JS) |
| 빌드 도구 | 없음 |
| 폰트 | Google Fonts — Jua, Nanum Gothic |
| AI 연동 | `toro_ai.js` (외부 스크립트) |
| 언어 | 한국어 UI |
